Bank Customer Churn Prediction 🚀📌 Project OverviewThis project aims to predict Customer Churn for a banking institution. By analyzing customer demographics and financial behavior, we developed a machine learning system that identifies customers who are likely to leave the bank. This allows the bank to take proactive retention measures.🛠️ Tech StackLanguage: Python 3.xLibraries: Pandas, NumPy, Scikit-Learn, Matplotlib, Seaborn, Plotly.Environment: Jupyter Notebook.📊 Dataset FeaturesThe model uses various customer attributes, including:Credit Score, Age, Tenure, and Balance.Number of Products, Credit Card ownership, and Active Membership status.Geography and Gender (processed via One-Hot Encoding).⚙️ Data Preprocessing & EngineeringData Cleaning: Removed irrelevant features like RowNumber, CustomerId, and Surname.Encoding: Applied One-Hot Encoding to categorical variables (Geography, Gender) with drop='first' to avoid the dummy variable trap.Feature Scaling: Utilized StandardScaler to normalize numeric features, ensuring models like SVM and KNN perform optimally.Handling Imbalance: Implemented class_weight='balanced' in tree-based models to handle the minority class (churned customers) effectively.🤖 Models & EvaluationI implemented and compared multiple machine learning algorithms:Random Forest (Best Performing)Support Vector Machine (SVM)Decision TreeK-Nearest Neighbors (KNN)Naive BayesModel Comparison Summary:ModelAccuracyPrecisionRecallF1-ScoreRandom Forest82.3%0.540.770.63SVM78.5%0.470.740.58Note: Priority was given to Recall, as capturing potential churners is more critical for the bank than overall accuracy.📈 Key Insights (Feature Importance)Based on the Random Forest model, the top factors driving customer churn are:Age: Older customers show a higher tendency to churn.NumOfProducts: Customers with a specific number of products are more likely to leave.IsActiveMember: Active members are significantly more loyal.
